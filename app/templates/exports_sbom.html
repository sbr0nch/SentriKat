    SBOM Export

    Software Bill of Materials. Export your full inventory as an industry-standard SBOM. Each format serves a different ecosystem: CycloneDX is the OWASP standard (widely adopted in AppSec/DevSecOps tooling), SPDX is the Linux Foundation / ISO 5962 standard (preferred for license compliance), STIX 2.1 is the OASIS standard for threat-intel sharing (ISACs, MISP, CISA). All three export the same underlying data for your organization only.
    CycloneDX 1.5

    JSON bundle with bomFormat: CycloneDX, specVersion: 1.5. Includes components[] with PURL and vendor, and vulnerabilities[] with CVSS ratings and affected refs. Compatible with Dependency-Track, Sonatype, GitHub Dependabot, Snyk, and any CycloneDX-aware SAST/DAST pipeline.

    /api/sbom/export/cyclonedx
    SPDX 2.3

    JSON bundle with spdxVersion: SPDX-2.3 and a packages[] array containing SPDXID, name, versionInfo, and externalRefs (CPE23Type) where available. Preferred by legal / procurement teams for software license compliance (ISO/IEC 5962).

    /api/sbom/export/spdx
    STIX 2.1

    JSON bundle with type: bundle and an objects[] array of vulnerability SDOs, software SCOs, and relationship SROs (affects). Consumed by ISACs, MISP, CISA AIS, and other threat-intel platforms.

    /api/sbom/export/stix21
    CLI / CI pipeline usage

    The same endpoints are callable via curl with your session cookie or an API key. This is the recommended path for CI/CD, dependency-track ingestion, and scheduled nightly exports. 

    # Session cookie (interactive shells)
curl -sk -H "Cookie: session=$SESSION" \
  https://app.sentrikat.com/api/sbom/export/cyclonedx -o sbom-cdx.json

# API key (recommended for CI)
curl -sk -H "X-API-Key: $SENTRIKAT_API_KEY" \
  https://app.sentrikat.com/api/sbom/export/spdx -o sbom-spdx.json
    Rate limit: 10 exports per hour per user. Exports larger than the per-bundle cap return HTTP 413 — filter with ?product_ids=1,2,3 or contact support for streaming exports.
